# 01. 디자인 브리프 — AeroOne 은 무엇인가

## 한 문장 정의

**AeroOne 은 인터넷이 차단된 사내 PC 에서도 그대로 돌아가는 뉴스레터·문서 열람 플랫폼** 입니다.

이미 발행된 HTML / PDF / Markdown 형식의 사내 뉴스레터를 한 곳에서 보고, ZIP 한 개로 인터넷이 끊긴 PC 에 동일하게 배포할 수 있는 modular monolith.

---

## 1. 왜 만들었는가

사내 뉴스레터는 매일 만들어지지만, **그것을 읽는 PC 는 보안상 인터넷이 차단된 환경** 입니다. 기존에는 발행자가 매번 PDF 파일을 메일로 돌리거나, 공유 폴더에 복사하거나, USB 로 옮겨야 했습니다. 문제는 다음 3 가지였습니다.

1. **다중 포맷 혼재** — 한 이슈가 HTML 본문 + PDF 인쇄본 + Markdown 노트로 동시에 존재. 사용자는 같은 이슈를 3 군데에서 찾아야 했습니다.
2. **검색·달력·태그가 없음** — 공유 폴더 방식은 "지난주 어떤 이슈가 있었는가" 를 빠르게 찾을 수 없었습니다.
3. **폐쇄망에서도 안전한 배포 경로 없음** — 인터넷이 가능한 PC 에서 만든 산출물을 폐쇄망 PC 로 옮기는 표준 절차가 없어, 매번 발행자가 수작업으로 동기화했습니다.

AeroOne 은 이 3 가지를 한 자리에서 해결합니다.

---

## 2. 누가 쓰는가

3 종류의 사용자가 있습니다. 우선순위 순서대로:

| 사용자 | 비중 | 주 행동 | 환경 |
|---|---|---|---|
| **뷰어 (사내 일반 직원)** | 95 % | 뉴스레터 목록 → 미리보기. 검색·달력·태그로 찾기. | 폐쇄망 Windows PC, Chrome / Edge |
| **관리자 (발행 담당자)** | 5 % | `Newsletter/output/` 폴더에 새 HTML/PDF 추가 → 관리자 페이지에서 import / sync. 메타데이터·태그·카테고리 정리. | 같은 폐쇄망 PC, 같은 브라우저, 단일 시드 관리자 계정 |
| **운영자 (IT 담당자)** | 1 회성 | 온라인 PC 에서 ZIP 빌드 → 폐쇄망 PC 에 풀고 batch 실행. 일반적으로 분기당 1~2 회. | Windows CMD / PowerShell |

운영자는 UI 를 거의 보지 않고 batch 스크립트로 일합니다. 본 디자인의 주 타겟은 **뷰어 (95 %)** 와 **관리자 (5 %)** 입니다.

---

## 3. 핵심 가치 (디자인이 지켜야 할 4 가지)

### 3.1 "한 이슈" 의 시각적 통일

한 이슈가 HTML + PDF + Markdown 3 자산으로 존재할 수 있습니다. 사용자는 그것을 **"하나의 카드"** 로 인식해야 합니다. 자산별로 카드가 3 개 떠 있으면 안 됩니다. 미리보기 화면 안에서 포맷 전환 (HTML ↔ PDF ↔ Markdown) 이 가능해야 합니다.

### 3.2 폐쇄망 적응 — 외부 의존 0

이 시스템은 **인터넷이 차단된 PC 에서 동작합니다**. 따라서 디자인이 외부 폰트, 외부 CDN, 외부 이미지, 외부 분석 스크립트를 참조하면 즉시 깨집니다. 이 제약은 절대적입니다 ([`04-design-constraints.md`](04-design-constraints.md) §1).

### 3.3 검색·달력·태그의 동등한 무게

뷰어가 이슈를 찾는 경로는 3 가지 — 키워드 검색, 달력 (발행일), 태그/카테고리. **이 세 가지는 모두 첫 화면에서 동일한 시각적 무게** 로 노출되어야 합니다. 한쪽이 부 (副) 의 위치에 있으면 다른 두 경로의 사용자가 길을 잃습니다.

### 3.4 관리자 화면도 같은 시스템 안에서

관리자 페이지는 별도 admin 디자인 시스템이 아니라, **같은 시각 언어 + 같은 컴포넌트** 위에 얹어야 합니다. 분량은 5 % 지만 매일 들어가는 화면이므로 이질감이 누적되면 피로도가 큽니다.

---

## 4. 디자인이 만들어야 할 정서

| 정서 | 의미 |
|---|---|
| **신뢰** | 사내 운영 시스템. 화려함보다 안정감. |
| **빠른 인지** | 첫 진입 3 초 안에 "오늘 새 이슈가 있는가 / 어제 이슈는 어디 있는가" 가 보여야 함. |
| **읽기 도구** | 본문이 주인공. UI 가 본문을 가리면 실패. |
| **건조한 우아함** | 기업 톤. 일러스트레이션 · 큰 hero · 마케팅적 카피 없음. |

---

## 5. 한 줄 요약

> AeroOne 의 디자인은 **"폐쇄망에서 외부 의존 0 으로 도는 / 라이트와 다크 양 모드를 동시에 운영하는 / HTML·PDF·Markdown 3 포맷을 하나의 카드로 묶는 / 검색·달력·태그를 동등하게 다루는 / 본문이 주인공이고 UI 가 보조인 / 5 % 관리자 화면도 같은 시각 언어를 쓰는"** 시스템입니다.
