# AeroOne Commit Writing Rule

이 저장소에서 커밋할 때는 아래 규칙을 항상 따른다.

## 커밋 작성 규칙

- 모든 커밋 메시지는 **한국어 제목 + 한국어 상세 본문**으로 작성한다.
- 제목은 변경의 의도를 드러내야 하며, 단순한 파일 나열이나 영문 축약형으로 끝내지 않는다.
- 본문은 짧게 끝내지 말고, 아래를 한국어로 충분히 남긴다.
  - 변경 배경
  - 선택한 접근 방식
  - 고려한 제약
  - 제외한 대안과 그 이유
- Lore protocol trailer 키는 유지하되, 각 trailer 값은 한국어로 구체적으로 적는다.

## 금지 사항

- 영문 한 줄 커밋
- 제목만 있고 본문이 없는 커밋
- 맥락 없이 `fix`, `update`, `cleanup`처럼 모호한 메시지
- 실제 의도보다 구현 세부만 나열한 메시지

## 권장 형식

- 제목: 한국어 한 줄
- 본문: 한국어 문단 1~3개
- trailer:
  - `Constraint: ...`
  - `Rejected: ...`
  - `Confidence: ...`
  - `Scope-risk: ...`
  - `Directive: ...`
  - `Tested: ...`
  - `Not-tested: ...`
