- 오프라인 ZIP wheel 중복: `pip download` 패키지당 단일 wheel dedup **실증**(신규 pypdf·python_docx·lxml·typing_extensions 각 1개).

## 3. 남은 후속 (각 단계 "후속" 취합)

- **F1**: LLM 보조 라우팅(규칙 미매칭 시).
- **F2**: 스트리밍 근거 합성(현재 비스트리밍 best-effort).
- **F3**: LLM 개조식 구조 생성(현재 규칙 포매터) · 미리보기 수정지시 · **승인 상태기계** · 종이(WASM) 미리보기 · **한컴 실기 HWPX 서식 호환 검증(운영자 한컴 PC 필요)**.
- **F4**: HWP(구 바이너리) 추출 · FTS5 트라이그램 · 상세검색 스트리밍 · 색인 품질/구조 보기.
- **F5**: 분류체계 마법사(LLM 업무 분류) · 주제 페이지 · 문서 카드(요약/품질) · 증분 이동 승계/소프트삭제.
- **F6**: 주/일 시간축 뷰 · 브라우저 토스트.
- **F7**: 대기 승인 패널(문서 승인 상태기계와 함께) · 헤더 배율.
- **F8**: **per-user LLM 프로필 전환**(사용자 선호 테이블 설계 필요 — 현재 환경설정은 관리자 연결 상태 read-only) · 성능 예산 · 브라우저 E2E.

## 4. 릴리스 권고

- **버전 스코프: major `2.0.0`** — 신모듈(aero_work) 다수 + 신규 마이그레이션 4개 + 신규 의존(pypdf·python-docx·lxml). AGENTS §9.6 상 minor/major 는 phase 보고서 필수(본 문서로 충족).
- **릴리스 전 선결 게이트**:
  1. 전체 백엔드 `pytest tests`(aero_work 외 회귀 포함) + 프런트 `vitest` 전량 green.
  2. **오프라인 ZIP 재빌드** — 신규 wheel(pypdf·python-docx·lxml) 반입 후 `offline_package.bat` 로 Task 5 검증 통과 확인(wheelhouse 단일 dir dedup은 실증됨).
  3. **한컴 실기 HWPX 호환 검증**(운영자 한컴 설치 PC) — 서식 확정 전 필수.
  4. 브라우저 E2E — `/aero-work` 7탭 + 오케스트레이션 실채팅 실측.
- **문서작성 승인 상태기계**는 릴리스 전 최소 구현 권장(gongmuwon '승인형 정책' §11 — 되돌리기 어려운 저장에 대기 승인).

## 5. 결론

중간 검토에서 "폭만 확보"였던 Aero Work 를, 실물 사용설명서를 진실 원천으로 삼아 **F1~F8 8개 단계로 깊이를 채웠다**. gongmuwon 정체성(대화→실무 오케스트레이션)과 문서 양식·지식위키·캘린더/알림·셸 UX 의 기본 구현을 갖췄고, 80 백엔드 테스트 + tsc 0 + 마이그레이션 가역 + wheel dedup 실증으로 검증했다. 남은 후속(승인 상태기계·LLM 구조 생성·분류체계·프로필 전환)과 릴리스 선결 게이트(한컴 실기·오프라인 ZIP 재빌드·전체 회귀)를 마치면 major(2.0.0) 릴리스가 가능하다.
